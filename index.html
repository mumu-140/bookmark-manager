使用说明

🔖 这个工具是做什么的？

一个整理浏览器书签的小工具，主要做三件事：把书签转换成各浏览器认得的格式、备份到云端、把两份书签对比合并成一份干净的。

全程在你自己的浏览器里运行，可以断网使用。书签数据不会上传到任何服务器——除非你主动点“上传到 Gist”，备份到你自己的 GitHub 账号。

🎯 我能用它解决什么？

  • 换浏览器：Safari 的书签想搬到 Chrome / Edge，格式不通用？转一下就行。
  • 备份书签：把书签存到 GitHub 云端，换电脑或重装系统后还能找回。
  • 合并去重：手机和电脑、两个浏览器的书签不一样，想对比后合并成一份。

📤 第一步：从浏览器导出书签

得先有一份书签文件，才能导入本工具。

  • Chrome / Edge：打开“书签管理器” → 右上角 ⋮ → 导出书签
  • Safari：菜单栏“文件” → 导出 → 书签
  • Firefox：“书签 → 管理书签” → 导入和备份 → 导出书签到 HTML

☁️ 功能一 · Gist 云端备份 / 转格式

  1. 导入一份书签文件（HTML 或 BookmarkHub 的 JSON）
  2. 选一个目标格式
  3. 点“💾 下载”存到本地，或点“☁️ 上传到 Gist”备份到云端

反过来也行：在“从 Gist 下载”里粘贴 Gist 链接，就能把云端书签取回来、再转成别的格式。

⚖️ 功能二 · 双向对比审核

导入 A、B 两份书签，工具自动找出它们的差异，你逐条决定怎么处理，最后导出合并好的结果。

  1. 分别导入 A、B 两份书签
  2. 点“开始对比审核”
  3. 逐条做决定，最后选格式导出

差异分三类：

  • A 独有 / B 独有：只在其中一份里出现的书签。
  • 共同差异：网址相同，但标题或所在文件夹不一样。

每条可以选的操作：

  • 保留 / 删除：留下或丢弃这一条。
  • → A / → B：把这条同步给缺它的那一侧。
  • 用 A / 用 B：网址相同时，统一采用 A 或 B 的标题和目录。

导出时，“审核后的合并结果”是按你的决策生成的；“原始文件”是某一侧的完整内容、不含任何修改。

📖 名词解释（看不懂的看这里）

书签 HTML 文件 — 浏览器“导出书签”生成的标准文件，扩展名 .html。几乎所有浏览器都能导入导出它，是书签搬家的通用中转格式。

GitHub Gist — GitHub 提供的“在线文本片段”服务，免费。本工具把它当云盘，用来存放书签文件；设为私有后只有你自己看得到。

Personal Access Token（PAT，访问令牌） — GitHub 的一把“专用钥匙”，上传到 Gist 时用它来证明是你本人。创建时只需勾选 gist 一项权限。本工具把它存在你浏览器本地，不会发给别人。
获取路径：GitHub → Settings → Developer settings → Personal access tokens。

BookmarkHub — 一个开源浏览器扩展，能把书签自动同步到 GitHub Gist。它备份出来的是 .txt 文件（内容其实是 JSON）。如果你在用这个扩展，就选 BookmarkHub.txt 格式。

🔒 隐私与安全

  • 整个工具就是单个网页文件，解析、转换都在你浏览器本地完成。
  • 不联网也能用；只有“上传 / 从 Gist 下载”时才会访问 GitHub。
  • 上传只会传到你自己的 GitHub 账号，默认是私有 Gist。
  • GitHub Token 只存在浏览器本地，可随时一键清除。
